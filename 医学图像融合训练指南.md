## ⚙️ 配置文件说明

配置文件: `configs/medical_fusion_config.yaml`

### 关键参数

#### 数据配置
```yaml
data:
  modality_a: "CT"          # 模态A (CT, T1-MRI, T2-MRI)
  modality_b: "T1-MRI"      # 模态B (CT, T1-MRI, T2-MRI)
  img_size: 256             # 图像尺寸
  train_ratio: 0.7          # 训练集比例
  val_ratio: 0.15           # 验证集比例
```

#### 训练配置
```yaml
training:
  epochs: 100               # 训练轮数
  batch_size: 4             # 批次大小 (GPU内存不足可改为2)
  learning_rate: 0.0001     # 学习率
  num_workers: 0            # Windows建议设为0
```

#### 损失函数权重
```yaml
loss:
  ssim_weight: 1.0          # SSIM损失权重
  gradient_weight: 10.0     # 梯度损失权重 (重要!)
  intensity_weight: 1.0     # 强度损失权重
```

## 🎯 训练建议

### 1. 首次训练 (快速测试)
修改配置文件:
```yaml
training:
  epochs: 10                # 先训练10个epoch测试
  batch_size: 2             # 减小batch_size避免内存不足
```

### 2. 正式训练
```yaml
training:
  epochs: 100               # 完整训练
  batch_size: 4             # 根据GPU内存调整
  early_stopping:
    enabled: true           # 启用早停
    patience: 20            # 20个epoch无改善则停止
```

### 3. 损失函数调优
如果梯度损失过高 (>2.0):
```yaml
loss:
  gradient_weight: 5.0      # 降低梯度损失权重
```

如果融合图像过于模糊:
```yaml
loss:
  gradient_weight: 15.0     # 增加梯度损失权重
```

### 4. 学习率调整
如果训练不稳定:
```yaml
training:
  learning_rate: 0.00005    # 降低学习率
  lr_scheduler: "plateau"   # 使用自适应学习率
```

## 🔧 常见问题

### 1. CUDA内存不足
```
RuntimeError: CUDA out of memory
```
**解决方案:**
- 减小 `batch_size` (4 → 2 → 1)
- 减小 `img_size` (256 → 128)
- 减小 `dim` (64 → 32)

### 2. 数据加载慢
**解决方案:**
- Windows: 保持 `num_workers: 0`
- Linux: 可以设置 `num_workers: 4`

### 3. 训练损失不下降
**可能原因:**
- 学习率过大或过小
- 损失函数权重不合理
- 数据归一化问题

**解决方案:**
- 调整学习率: `0.0001` → `0.00005`
- 检查数据加载是否正常
- 查看TensorBoard损失曲线

### 4. 梯度爆炸
```
RuntimeError: Function 'XXX' returned nan values
```
**解决方案:**
- 启用梯度裁剪: `grad_clip: 1.0`
- 降低学习率
- 检查数据是否有异常值

## 📈 评估指标说明

### 1. 损失指标
- **Total Loss**: 总损失 (越小越好)
- **SSIM Loss**: 结构相似性损失 (越小越好)
- **Gradient Loss**: 梯度损失 (越小越好，参考值: <0.5优秀, 0.5-1.0良好)
- **Intensity Loss**: 强度损失 (越小越好)

### 2. 融合质量指标
- **EN (信息熵)**: 越大越好 (>7.0优秀)
- **SF (空间频率)**: 越大越好 (>10.0优秀)
- **SSIM**: 结构相似性 (0-1, 越大越好, >0.8优秀)
- **MI (互信息)**: 越大越好 (>1.5优秀)
